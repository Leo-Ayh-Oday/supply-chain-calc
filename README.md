## 这是什么

一个给采购经理用的计算工具，回答三个问题：

- **如果某家供应商突然停产，你的工厂能撑几天？**
- **三家备选供应商哪家最划算？**
- **万一断供，到底会亏多少钱？**

填几个数字，秒出结果。每条结论附带完整审计链，可以追溯到原始数据——不只是"我觉得"，是"数据说"。

## 核心算法

```
存活天数 = 当前库存 / 日消耗量
停产天数 = max(0, 备选到货周期 - 存活天数)
停产损失 = 停产天数 × 停工日罚金
切换成本 = 日消耗 × 安全库存天数 × 备选单价
订单违约金 = max(0, 到货周期 - 存活天数) × 违约金率
总损失 = 停产损失 + 切换成本 + 订单违约金
```

按总损失升序排列，最低即为推荐方案。

**代码全部可查。** 不信我的公式？打开 `index.html` 搜索 `function simulate`——整个引擎不到 60 行，没有黑箱。

## 最终产品规划（在线版）

等经费到位后，在线版将升级为一个完整的供应链压力测试平台。后端已开发完成，界面如下：

### 完整功能

**🔐 账号系统 & 邀请制裂变**

注册即可获得 3 次免费推演。每邀请一位同事，双方各 +1 次额度。形成"用的人越多，每个人额度越多"的正循环。

**📊 多场景仪表盘**

不再是单次计算。保存所有推演场景——"江苏供应商断供"、"两家同时出事"、"库存减半会怎样"——随时切换对比。

**🔗 完整审计链**

每条结论都可追溯到原始输入数据，基于 Antigone 确定性引擎的因果树推演。不只是"选广东大厂损失 38.5 万"，而是"为什么选广东——因为到货 22 天仅停产 4 天，且单价 85 元在三个备选中最低，总损失 = 停产 24 万 + 切换 8.5 万 + 违约 6 万"。老板问起来，直接导出 PDF 报告发过去。

**📈 PDF/Excel 专业报告**

一键导出带排版、带图表、带审计链的正式报告。可以直接附在采购申请单后面。

**🤖 LLM 叙事摘要**

基于 DeepSeek 等大模型，将枯燥的数字自动转化为管理层能看懂的叙事——"当前库存水平下，广东大厂为最优方案。若库存进一步下降至 500 件以下，本地小厂的到货速度优势将凸显。建议保持安全库存不低于 15 天。"

**🌐 多供应商同时断供**

不只算一个供应商出事——两个同时断、三个连环断，多级供应链推演。

### 后端技术栈

- **确定性引擎**：Antigone ConstraintSolver（WorldDef 规则定义 → 约束求解 → 因果树解释）
- **鉴权**：bcrypt + JWT + SQLite（零外部依赖）
- **叙事报告**：DeepSeek API（无 API Key 时降级为确定性公式摘要）
- **部署**：FastAPI + Nginx 反向代理，2 核 2G 即可承载数百并发
